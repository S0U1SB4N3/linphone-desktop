\subsubsection{L\+I\+N\+P\+H\+O\+N\+E\+\_\+\+T\+U\+N\+N\+EL}
{\footnotesize\ttfamily \#define L\+I\+N\+P\+H\+O\+N\+E\+\_\+\+T\+U\+N\+N\+EL(\begin{DoxyParamCaption}\item[{}]{obj }\end{DoxyParamCaption})~B\+E\+L\+L\+E\+\_\+\+S\+I\+P\+\_\+\+C\+A\+ST(obj, \textbf{ Linphone\+Tunnel})}



Linphone tunnel aims is to bypass IP traffic blocking due to aggressive firewalls which typically only authorize T\+CP traffic with destination port 443. 

~\newline
 Its principle is tunneling all S\+IP and/or R\+TP traffic through a single secure https connection up to a detunnelizer server. ~\newline
 This set of methods enhance Linphone\+Core functionalities in order to provide an easy to use A\+PI to \begin{DoxyItemize}
\item provision tunnel servers IP addresses and ports. This functionality is an option not implemented under G\+PL. Availability can be check at runtime using function \doxyref{linphone\+\_\+core\+\_\+tunnel\+\_\+available}{p.}{group__tunnel_ga6c411bd114b1e45806dbbb7c02912b35} \item start/stop the tunneling service \item perform auto-\/detection whether tunneling is required, based on a test of sending/receiving a flow of U\+DP packets.\end{DoxyItemize}
It takes in charge automatically the S\+IP registration procedure when connecting or disconnecting to a tunnel server. No other action on Linphone\+Core is required to enable full operation in tunnel mode.

~\newline
 Provision is done using object \#\+Linphone\+Tunnel\+Config created by function \doxyref{linphone\+\_\+tunnel\+\_\+config\+\_\+new()}{p.}{group__tunnel_ga779c4280b7f6c1e5a161fd98e6671d81}. Functions \doxyref{linphone\+\_\+tunnel\+\_\+config\+\_\+set\+\_\+host}{p.}{group__tunnel_ga3528ad019e02f1dafbed28b588032309} and \doxyref{linphone\+\_\+tunnel\+\_\+config\+\_\+set\+\_\+port}{p.}{group__tunnel_ga01f6ec956efbb074a611c4256e86eab0} allow to point to tunnel server I\+P/port. Once set, use function \doxyref{linphone\+\_\+tunnel\+\_\+add\+\_\+server}{p.}{group__tunnel_ga762cc31830da8eb5736017e690ced895} to provision a tunnel server. ~\newline
 Finally tunnel mode configuration is achieved by function \doxyref{linphone\+\_\+tunnel\+\_\+set\+\_\+mode}{p.}{group__tunnel_ga2ff9779aefdb5455c3509d75cf89a88c}. ~\newline
 Tunnel connection status can be checked using function \doxyref{linphone\+\_\+tunnel\+\_\+connected}{p.}{group__tunnel_ga047ca9bc9546a6c5bffb27209b0a7706}.

Bellow pseudo code that can be use to configure, enable, check state and disable tunnel functionality\+:
